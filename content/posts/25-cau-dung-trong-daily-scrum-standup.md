## Giới thiệu

Daily Scrum (hay Standup Meeting) là buổi họp ngắn **15 phút** mỗi ngày trong Agile team. Ba câu hỏi kinh điển:

1. **What did you do yesterday?** — Hôm qua làm gì?
2. **What will you do today?** — Hôm nay sẽ làm gì?
3. **Any blockers?** — Có gì bị chặn không?

Nghe đơn giản, nhưng nếu bạn không có sẵn "kho câu", bạn sẽ lặp đi lặp lại mấy câu nhàm chán. Bài này tổng hợp **25 mẫu câu thực tế** giúp bạn nói standup tự tin, chuyên nghiệp và đa dạng hơn! 🎯

## 1. Báo cáo việc đã làm (Yesterday / What I did)

| # | Tiếng Anh | Tiếng Việt |
|---|-----------|------------|
| 1 | Yesterday, I **completed** the user authentication module. | Hôm qua, tôi **hoàn thành** module xác thực người dùng. |
| 2 | I **finished up** the code review for PR #234. | Tôi **hoàn tất** code review cho PR #234. |
| 3 | I **spent most of the day** debugging the payment integration issue. | Tôi **dành phần lớn thời gian** debug lỗi tích hợp thanh toán. |
| 4 | I **wrapped up** the unit tests for the search service. | Tôi **hoàn tất** unit test cho search service. |
| 5 | I **paired with** [name] on the database migration script. | Tôi **pair programming với** [tên] về script migration database. |

### 💡 Mẹo nói tự nhiên

- Dùng **completed / finished up / wrapped up** thay vì chỉ nói "I did..."
- Thêm chi tiết cụ thể: số PR, tên module, tên feature
- **Paired with** cho thấy bạn có collaborate với team

## 2. Báo cáo kế hoạch hôm nay (Today / What I'll do)

| # | Tiếng Anh | Tiếng Việt |
|---|-----------|------------|
| 6 | Today, I'm **going to work on** the notification system. | Hôm nay, tôi sẽ **làm việc về** hệ thống thông báo. |
| 7 | I'll **pick up** ticket PROJ-456 — the CSV export feature. | Tôi sẽ **nhận** ticket PROJ-456 — tính năng export CSV. |
| 8 | I'm **planning to** finish the API documentation by EOD. | Tôi **dự định** hoàn thành API documentation trước cuối ngày. |
| 9 | I'll **continue working on** the performance optimization from yesterday. | Tôi sẽ **tiếp tục** tối ưu performance từ hôm qua. |
| 10 | My main focus today is **getting the staging environment ready** for QA. | Mục tiêu chính hôm nay là **chuẩn bị staging environment** cho QA. |

### 💡 Mẹo nói tự nhiên

- **Pick up** (nhận ticket) rất phổ biến trong Agile team
- **By EOD** = By End of Day — cụm từ hay dùng
- **My main focus today is...** nghe chuyên nghiệp hơn "I will do..."

## 3. Nêu Blockers / Vấn đề gặp phải

| # | Tiếng Anh | Tiếng Việt |
|---|-----------|------------|
| 11 | I'm **blocked on** the API integration — waiting for the endpoint from the backend team. | Tôi bị **chặn ở** API integration — đang chờ endpoint từ team backend. |
| 12 | I **ran into** a dependency issue with the new library version. | Tôi **gặp phải** vấn đề dependency với phiên bản thư viện mới. |
| 13 | I **need access to** the production logs to investigate the bug. | Tôi **cần quyền truy cập** production logs để điều tra bug. |
| 14 | There's a **flaky test** in the CI pipeline that keeps failing intermittently. | Có một **test không ổn định** trong CI pipeline cứ fail ngắt quãng. |
| 15 | I'm **stuck on** a race condition in the messaging queue — might need a second pair of eyes. | Tôi **bị kẹt** ở race condition trong messaging queue — có thể cần người review giúp. |

### 💡 Mẹo nói tự nhiên

- **Blocked on** là cách nói chuẩn trong Agile
- **Ran into** = gặp phải (informal, tự nhiên)
- **A second pair of eyes** = nhờ ai đó review — rất idiomatic!

## 4. Xin hỗ trợ / Phối hợp

| # | Tiếng Anh | Tiếng Việt |
|---|-----------|------------|
| 16 | Could I **sync up with** [name] after standup about the schema changes? | Tôi có thể **sync với** [tên] sau standup về thay đổi schema không? |
| 17 | I'll **need** [name]'s **input on** the caching strategy before I proceed. | Tôi sẽ **cần ý kiến** của [tên] **về** chiến lược caching trước khi tiến hành. |
| 18 | Can someone from the DevOps team **help me set up** the monitoring dashboard? | Ai bên team DevOps có thể **giúp tôi setup** monitoring dashboard không? |
| 19 | I'd like to **get a quick review** on my PR before merging — it touches the auth flow. | Tôi muốn **nhờ review nhanh** PR trước khi merge — nó liên quan đến auth flow. |
| 20 | Let's **take this offline** — I'll set up a meeting to discuss the architecture. | Mình **bàn riêng** nhé — tôi sẽ book meeting để thảo luận kiến trúc. |

### 💡 Mẹo nói tự nhiên

- **Sync up with** = họp nhanh, trao đổi
- **Take this offline** = bàn ngoài standup (vì standup chỉ 15 phút!)
- **Get a quick review** — lịch sự và rõ ràng

## 5. Cập nhật tiến độ & Điều phối (Scrum Master / Tech Lead)

| # | Tiếng Anh | Tiếng Việt |
|---|-----------|------------|
| 21 | Just a **heads up** — the sprint deadline is Friday, so let's prioritize the remaining tickets. | Lưu ý nhé — deadline sprint là thứ Sáu, nên hãy ưu tiên các ticket còn lại. |
| 22 | I **made good progress** on the migration, about **80% done**. | Tôi **tiến triển tốt** về migration, khoảng **80% xong**. |
| 23 | The feature is **on track** for release next Tuesday. | Feature đang **đúng tiến độ** để release thứ Ba tới. |
| 24 | I **underestimated** the complexity — I'll need **one more day** to finish. | Tôi **đánh giá thấp** độ phức tạp — sẽ cần **thêm 1 ngày** để hoàn thành. |
| 25 | **No blockers** from my side — everything is going smoothly. | **Không có blocker** phía tôi — mọi thứ đang suôn sẻ. |

### 💡 Mẹo nói tự nhiên

- **Heads up** = thông báo trước, nhắc nhở
- **On track** = đúng tiến độ
- **Underestimated** — thừa nhận đánh giá sai, rất professional (tốt hơn là bào chữa!)

## 7. Script mẫu — Một buổi Standup hoàn chỉnh

> **Scrum Master:** Alright, let's get started. Minh, you're up first.
>
> **Minh:** Sure! Yesterday, I **wrapped up** the unit tests for the payment service and **paired with** Lan on the database migration. Today, I'll **pick up** PROJ-789 — the email notification feature. **No blockers** from my side.
>
> **Scrum Master:** Great. Lan?
>
> **Lan:** Yesterday, I **spent most of the day** debugging the WebSocket connection issue. I **ran into** a race condition that's hard to reproduce. Today, I'll **continue working on** it. I'm **not blocked**, but I **might need a second pair of eyes** — Minh, could we **sync up** after standup?
>
> **Minh:** Sure thing!
>
> **Scrum Master:** David?
>
> **David:** Yesterday, I **finished up** the API documentation. Today, my **main focus** is **getting the staging environment ready** for QA. I'm **blocked on** one thing — I **need access to** the AWS console. Could someone from DevOps **help me set up** the permissions?
>
> **Scrum Master:** I'll connect you with Thao right after. Just a **heads up** everyone — sprint deadline is Friday. Let's make sure everything is **on track**.

## 8. Bài tập thực hành

### Exercise 1: Viết standup update

Bạn là developer. Hôm qua bạn fix xong bug login, hôm nay bạn sẽ làm feature upload file, và bạn đang chờ design từ team UI. Viết standup update bằng tiếng Anh.

<details>
<summary>📝 Đáp án gợi ý</summary>

"Yesterday, I **completed** the login bug fix — it was a session timeout issue. Today, I'm **going to work on** the file upload feature, ticket PROJ-123. I'm **blocked on** the UI design — still waiting for the mockups from the design team. Could we **sync up** with the design team to get an ETA?"

</details>

### Exercise 2: Nêu blocker chuyên nghiệp

Bạn gặp lỗi test CI cứ fail ngẫu nhiên, không biết nguyên nhân. Viết câu nêu blocker.

<details>
<summary>📝 Đáp án gợi ý</summary>

"I **ran into** a **flaky test** in the CI pipeline — it passes locally but **fails intermittently** on the CI server. I've **spent** about 2 hours investigating but haven't been able to reproduce it consistently. I **might need a second pair of eyes** on this — anyone familiar with the test runner config?"

</details>

### Exercise 3: Xin hỗ trợ

Bạn cần review PR gấp vì feature phải ship hôm nay. Viết câu xin review.

<details>
<summary>📝 Đáp án gợi ý</summary>

"I'd like to **get a quick review** on PR #567 — it's the checkout flow refactor. It needs to ship today, so if anyone has **15 minutes** to take a look, that would be great. I'll **take this offline** after standup if needed."

</details>

## Tổng kết

Daily Standup không cần câu phức tạp — chỉ cần **rõ ràng, ngắn gọn, và đúng trọng tâm**. Với 25 mẫu câu trên, bạn có thể:

- ✅ Báo cáo đa dạng, không lặp lại
- ✅ Nêu blocker chuyên nghiệp
- ✅ Xin hỗ trợ lịch sự
- ✅ Nghe tự nhiên như native speaker

**Mẹo cuối:** Chuẩn bị standup trước 5 phút — ghi nhanh 3 bullet points. Bạn sẽ tự tin hơn nhiều! 💪
