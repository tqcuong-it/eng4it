## 📰 Bài đọc (English)

Kubernetes, which was originally developed by Google, has become the {{< vocab w="standard" ipa="/ˈstændərd/" vn="tiêu chuẩn" t="noun" >}} for {{< vocab w="container orchestration" ipa="/kənˈteɪnər ˌɔːrkɪˈstreɪʃən/" vn="điều phối container" t="noun" >}}. A recent report shows that over 80% of companies that use {{< vocab w="containers" ipa="/kənˈteɪnərz/" vn="container" t="noun" >}} have adopted Kubernetes for their {{< vocab w="workloads" ipa="/ˈwɜːrkˌloʊdz/" vn="khối lượng công việc" t="noun" >}}.

{{< vocab w="DevOps" ipa="/ˈdevɑːps/" vn="DevOps" t="noun" >}} teams who need to {{< vocab w="deploy" ipa="/dɪˈplɔɪ/" vn="triển khai" t="verb" >}} applications at {{< vocab w="scale" ipa="/skeɪl/" vn="quy mô lớn" t="noun" >}} rely on Kubernetes to manage thousands of containers automatically. If a container {{< vocab w="crashes" ipa="/ˈkræʃɪz/" vn="bị lỗi, sập" t="verb" >}}, Kubernetes will {{< vocab w="restart" ipa="/riːˈstɑːrt/" vn="khởi động lại" t="verb" >}} it without any human {{< vocab w="intervention" ipa="/ˌɪntərˈvenʃən/" vn="sự can thiệp" t="noun" >}}.

All major cloud providers, which include AWS, Azure, and Google Cloud, now offer {{< vocab w="managed" ipa="/ˈmænɪʤd/" vn="được quản lý" t="adj" >}} Kubernetes services. If your team adopts Kubernetes, it will be easier to move workloads between different cloud platforms.

## 🇻🇳 Bản dịch tiếng Việt

Kubernetes, vốn được phát triển bởi Google, đã trở thành tiêu chuẩn cho việc điều phối container. Một báo cáo gần đây cho thấy hơn 80% công ty sử dụng container đã áp dụng Kubernetes cho khối lượng công việc của họ.

Các nhóm DevOps cần triển khai ứng dụng ở quy mô lớn dựa vào Kubernetes để quản lý hàng nghìn container tự động. Nếu một container bị sập, Kubernetes sẽ khởi động lại nó mà không cần sự can thiệp của con người.

Tất cả nhà cung cấp cloud lớn, bao gồm AWS, Azure và Google Cloud, hiện đều cung cấp dịch vụ Kubernetes được quản lý. Nếu nhóm của bạn áp dụng Kubernetes, việc di chuyển khối lượng công việc giữa các nền tảng cloud khác nhau sẽ dễ dàng hơn.

## 📝 Phân tích ngữ pháp

### Câu 1: "Kubernetes, which was originally developed by Google, has become the standard for container orchestration."
- **Cấu trúc:** S + , which + was V(past participle) + by + O + , + has become + O
- **Ngữ pháp:** Mệnh đề quan hệ không xác định với "which" ở dạng bị động quá khứ
- **Ví dụ tương tự:** Docker, which was released in 2013, changed how developers deploy applications.

### Câu 2: "DevOps teams who need to deploy applications at scale rely on Kubernetes."
- **Cấu trúc:** S + who + need to + V + O + V + on + O
- **Ngữ pháp:** Mệnh đề quan hệ xác định với "who" + "rely on" (dựa vào)
- **Ví dụ tương tự:** Companies who want to reduce costs rely on cloud computing.

### Câu 3: "If a container crashes, Kubernetes will restart it without any human intervention."
- **Cấu trúc:** If + S + V(present), S + will + V + O + without + O
- **Ngữ pháp:** Câu điều kiện loại 1 (First conditional) — mô tả hành vi tự động
- **Ví dụ tương tự:** If the server goes down, the load balancer will redirect traffic.

### Câu 4: "All major cloud providers, which include AWS, Azure, and Google Cloud, now offer managed Kubernetes services."
- **Cấu trúc:** S + , which + V + O + , + adv + V + O
- **Ngữ pháp:** Mệnh đề quan hệ không xác định với "which" — liệt kê các nhà cung cấp
- **Ví dụ tương tự:** The big three browsers, which include Chrome, Firefox, and Safari, all support WebAssembly.

## ✏️ Bài tập

### Comprehension (Đọc hiểu)
1. Kubernetes được phát triển ban đầu bởi công ty nào?
2. Kubernetes làm gì khi một container bị sập?
3. Lợi ích của việc áp dụng Kubernetes là gì?

### Vocabulary (Từ vựng)
Điền từ thích hợp:
1. We need to ___ the new version of the app to production servers.
2. Kubernetes has become the ___ for container management.
3. If the service ___, the system will automatically recover.
4. The team handles heavy ___ during peak hours.

{{< spoiler "✅ Đáp án" >}}
**Comprehension:**
1. Google phát triển Kubernetes ban đầu.
2. Kubernetes tự động khởi động lại (restart) container mà không cần sự can thiệp của con người.
3. Dễ dàng di chuyển workloads giữa các nền tảng cloud khác nhau.

**Vocabulary:**
1. **deploy** — triển khai
2. **standard** — tiêu chuẩn
3. **crashes** — bị sập
4. **workloads** — khối lượng công việc
